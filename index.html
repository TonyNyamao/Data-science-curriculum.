Mentor guide — how to use this

Step 01
Student attempts each query independently before revealing the hint. The attempt — even if wrong — is the learning moment.
Step 02
Debug together. Ask the student to explain each clause. Confusion on WHERE vs HAVING, or JOIN type, surfaces naturally.
Step 03
For JOIN questions, have the student draw the table relationships on paper first. Visual mapping prevents most mistakes.
Step 04
Phase 5 is a project presentation. Each query is a dashboard deliverable. Student should explain what it means for the business, not just that it runs.
